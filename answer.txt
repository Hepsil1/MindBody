# Відповіді на технічні питання (Antigravity AI)

## 1) Поточний фронтенд
*   **Стек**: Чистий HTML + Vanilla JS. Це не Next.js або React. 
*   **Рендеринг**: Клієнтський (CSR). Браузер вантажить статику, JS забирає дані з JSON і малює їх.
*   **Роутинг**: Файловий (`index.html`, `shop-women.html`). Використовуються URL параметри (наприклад, `product.html?id=ID`). Справжнього SPA-роутера немає.

## 2) Головна сторінка
*   **Статус**: Зараз `index.html` — статичний файл.
*   **Конструктор**: Так, ми можемо замінити частину контенту на динамічний рендер з JSON без втрати дизайну.
*   **Впровадження**: Найкраще створити `HomePage.js`, який буде читати конфіг (секції) і викликати відповідні CSS-блоки.

## 3) Бекенд та база даних
*   **Статус**: Зараз бекенда немає (замінено моками в `api.js` та `localStorage`).
*   **Рекомендація**: 
    *   **FastAPI (Python)** — якщо хочете сучасний, дуже швидкий API та легкість розробки. (Це було в початковому плані).
    *   **Django** — якщо хочете "все з коробки", включаючи потужну адмінку за 5 хвилин.
*   **База**: **PostgreSQL** — ідеальний стандарт. Для локальної розробки почнемо з SQLite.

## 4) Адмінка
*   **Підхід**: Я рекомендую **Варіант А** (окремий фронтенд на `/admin`). Це дозволить зробити сучасний UI (на React + shadcn/ui), який не залежить від коду основного магазину.
*   **UI Kit**: Краще взяти **shadcn/ui** або **MUI**, щоб не витрачати час на малювання кнопок з нуля.

## 5) Авторизація
*   **Вхід**: Email + Password.
*   **Сесія**: JWT у HTTP-only cookies (найбільш безпечно).
*   **2FA**: Для MVP не обов'язково, але в майбутньому варто додати через Telegram-боти.

## 6) Товари
*   **Варіанти**: Обов'язково (Розмір + Колір).
*   **Склад**: Поки достатньо статусу "в наявності", але краще закласти поле "кількість".
*   **Структура**: Назва, ціна, опис, фото (галерея), категорія, колекція, склад/догляд, таблиця розмірів.

## 7) Замовлення
*   **Процес**: Замовлення створюються клієнтами на сайті, а адмін може створювати їх вручну.
*   **Статуси**: Нове, В обробці, Відправлено, Доставлено, Скасовано.
*   **Звіти**: Виручка, кількість замовлень, топ-товари — це мінімум для старту.

## 8) Зображення
*   **Збереження**: Локально в `/uploads` на VPS. CDN (Cloudflare) підключимо пізніше.
*   **Обробка**: Автоматичне стиснення та конвертація у WebP — обов'язково для швидкості.
*   **Ліміти**: До 10MB, формати: JPG, PNG, WEBP.

## 9) Конструктор (Index Builder)
*   **Секції**: Hero, Категорії, Новинки, Переваги (Values), Instagram, Newsletter.
*   **Функції**: Drag&drop та Draft/Publish — дуже корисні, варто впроваджувати одразу.
*   **Preview**: Iframe всередині адмінки — підтверджуємо.

## 10) Деплой
*   **Структура**: Однозначно **Docker**. Це дозволить легко переносити проект між серверами.
*   **Конфіг**: `.env` для всіх секретів та паролів. Паролі в БД — тільки в хешованому вигляді (BCrypt).
